# skills-manager project overview

- Purpose: `skls-mgr` is a Node.js CLI for centrally managing agent skills in `~/.config/skls-mgr`, then installing them into projects on demand.
- Core workflow:
  - `add`: ingest skills from GitHub/git/local sources into the central base dir.
  - `install`: copy or symlink selected skills into a target project directory.
  - `list`: show managed and manual skills in the base dir.
  - `update`: refresh previously tracked skills.
  - `remove`: delete installed skills from the base dir.
- Tech stack: TypeScript, Node.js >= 22, ESM (`type: module`), `pnpm`, Vitest, `tsx` for local development.
- Main runtime dependencies: `@clack/prompts` for interactive CLI, `picocolors` for terminal colors, `gray-matter` for frontmatter parsing, `simple-git` for git operations.
- Codebase structure:
  - `src/cli.ts`: CLI entrypoint and command dispatch.
  - `src/add.ts`, `src/install.ts`, `src/update.ts`, `src/remove.ts`, `src/list.ts`: command implementations.
  - `src/source-parser.ts`, `src/skills.ts`, `src/skill-lock.ts`, `src/filesystem.ts`, `src/git.ts`, `src/paths.ts`, `src/base-dir.ts`: core support modules.
  - `src/i18n.ts`, `src/prompt-format.ts`, `src/constants.ts`, `src/types.ts`: shared infra and types.
  - `bin/cli.mjs`: published bin shim importing `dist/cli.js`.
  - `tests/*.test.ts`: Vitest test suite, including CLI integration coverage.
- Build output: `dist/` generated by `tsc -p tsconfig.build.json`.
- Repository remote: `git@github.com:Xaviw/skills-manager.git`.
- Platform notes: onboarding happened on Darwin (macOS). Standard shell utilities expected: `git`, `ls`, `cd`, `find`, `rg`/`grep`, `sed`.
